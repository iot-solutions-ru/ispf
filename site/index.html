ISPF

Self-hosted industrial IoT / SCADA — devices, HMI, alarms, historian, and BPMN on one object tree.

Quick start Documentation

One model for the plant

Most stacks glue OPC, historian, HMI, alarms, and workflow as separate products. ISPF keeps them as nodes on the same hierarchical tree — same API, same permissions, same UI.

  • Object tree Typed variables, events, and functions with REST and WebSocket access.
  • HMI & SCADA Dashboard builder, charts with history, and mimic diagrams for operators.
  • Automation CEL alert rules, correlators, and BPMN workflows as first-class tree nodes.
  • Drivers & apps ~58 protocol drivers and declarative application bundles — solutions without forking the core.

Built for operators and builders

Admin Explorer for the model. Operator mode for the floor. Screenshots from the live demostand.

Run it locally

Portable zip with embedded H2, or pull the container. Default login is admin / admin.

docker pull ghcr.io/michaael/ispf-server:latest
docker run --rm -p 8080:8080 -v ispf-data:/opt/ispf/data ghcr.io/michaael/ispf-server:latest
# open http://localhost:8080
