ISPF

Self-hosted промышленная IoT / SCADA — устройства, HMI, аварии, историк и BPMN на одном дереве объектов.

Быстрый старт Документация

Одна модель завода

Обычные стеки склеивают OPC, историк, HMI, аварии и workflow как отдельные продукты. В ISPF это узлы одного иерархического дерева — один API, одни права, один UI.

  • Дерево объектов Типизированные переменные, события и функции через REST и WebSocket.
  • HMI и SCADA Конструктор дашбордов, графики с историей и мнемосхемы для операторов.
  • Автоматизация Правила аварий на CEL, корреляторы и BPMN-процессы как узлы дерева.
  • Драйверы и приложения ~58 протокольных драйверов и декларативные бандлы — решения без форка ядра.

Для операторов и разработчиков

Admin Explorer для модели. Operator mode для цеха. Кадры с живого демостенда.

Запуск локально

Portable zip со встроенным H2 или контейнер. Логин по умолчанию: admin / admin.

docker pull ghcr.io/michaael/ispf-server:latest
docker run --rm -p 8080:8080 -v ispf-data:/opt/ispf/data ghcr.io/michaael/ispf-server:latest
# открыть http://localhost:8080
