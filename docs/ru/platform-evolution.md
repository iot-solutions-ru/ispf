# Эволюция ISPF — ретроспективный бэклог

Хронологический контрольный список того, **что было сделано** во время существования платформы. Все пункты отмечены как выполненные — это не план, а **ретроспектива** развития.

**Базовый уровень:** `main` → `0.9.41`.  
**Источники:** история git, [ROADMAP.md](roadmap.md), релизные теги.

---

## Как читать документ

| Символ | Значение |
|--------|----------|
| `[x]` | Реализовано и в baseline `main` |
| `[~]` | Частично / in progress (см. [ROADMAP.md](roadmap.md)) |
| `[ ]` | Запланировано, ещё не закрыто |

Эволюция идет **сверху вниз** — от фундамента к современному состоянию.

---

## Вехи решения

| Версия | Смысл |
|--------|-------|
| **v0.1.x** | Первая работоспособная базовая версия: дерево объектов, драйверы, дашборды |
| **v0.2.0** | Закрытие фазы 5 — декларативные цепочки, модели, конвергенция пакетов |
| **v0.3.0** | Готовность производства: федерация, зрелость драйвера, ПФ-09/11 |
| **v0.5.x** | Наложение привязки Федерации, исходящий туннель |

## 2. Платформа приложений — REQ-PF

Поворотный момент: решения размещаются **на платформе** посредством развертывания пакета, а не в отраслевом Java-сервере.

- [x] **Спецификация REQ-PF** — [ROADMAP.md § Часть A](roadmap.md), ADR [0001](decisions/0001-app-platform-boundary.md)
- [x] **Sprint A** — время выполнения скрипта: `selectMany`, `setVar`, `when`/`if`, передача; Схема приложения изоляции (PF-02)
- [x] **Sprint B** — метаданные пакета, профиль провода BFF `ispf-operator-v1`, `cancel_workflows` (PF-06, PF-10)
- [x] **Sprint C** — manifest-driven operator shell через generic `POST /bff/invoke`
- [x] **Sprint C P2** — привязки SQL, откат функций, профили виртуального симулятора (PF-08, PF-09)
- [x] **Sprint D** — откат пакета, API манифеста оператора, тесты четности терминала
- [x] **API развертывания приложений** — `POST /applications/{appId}/deploy`: миграции, функции, объекты, дашборды, рабочие процессы, модели.
- [x] **Platform scheduler** — cron через `platform_schedules`, не `@Scheduled` в Java (PF-05)
- [x] **Script functions** — JSON steps + SQL, без custom `FunctionHandler` в `main` (PF-01)
- [x] **Справочные приложения** — `examples/demo-app`, `warehouse-app` (dogfoodinggate [0002](decisions/0002-dogfooding-gate.md))

## 19. Принципы, которые не менялись

Эти решения прошли через всю эволюцию:

- [x] **North star** — бизнес-логика в конфигурации дерева объектов, не в отраслевом Java ([ARCHITECTURE.md](architecture.md))
- [x] **Dogfoodinggate** — каждый REQ-PF создается из создателя app-команды ([0002](decisions/0002-dogfooding-gate.md))
- [x] **Tree-first** — правила оповещений, корреляторы, отчеты, расписания, безопасность — узлы дерева
- [x] **Bundle = упаковка** — развертывание создает конфигурацию на платформе, независимо от времени выполнения.
- [x] **Generic platform / industry solution** — разделение `main` vs `examples/*`
