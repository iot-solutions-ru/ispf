# Партнерская программа (BL-184)

Сертифицированный ISPF интегратор и партнерская программа по решениям. **Партнерский портал** — отдельный сервис:

- Repo: [Partner-portal](https://github.com/your-org/Partner-portal)
- API: `GET /api/v1/tiers`, applications, directory, training progress
- Внутрисерверный API `GET /api/v1/partners/*` — **заглушка** (`PartnerProgramService`, `"source": "stub"`); демо-партнеры жестко запрограммированы; регистрация является синтетической до тех пор, пока партнерский портал не синхронизируется.

Связано: [marketplace](marketplace.md), [certification](certification.md), [solution-developer-guide](solution-developer-guide.md).

## Уровни сертификации

| Уровень | Аудитория | Предпосылки | Проверяет |
|-------|----------|---------------|-----------|
| **Сотрудник** | Разработчики, впервые работающие с ISPF | Курс по основам платформы | Дерево объектов, пакеты, развертывание пользовательского интерфейса оператора |
| **Профессионал** | Интеграторы решений | Партнер + 1 реализованный проект | Драйверы, дашборды, автоматизация, основы федерации |
| **Эксперт** | Ведущие архитекторы / OEM-партнеры | Профессионал + 2 производственные площадки | Кластер, уровни архиватора, пакет MES, сценарии для агентов |
| **ОЕМ** | Поставщики продукции (драйверы, символы, analytics KPI packs) | Юридическая лаборатория + лаборатория взаимодействия | Подписание пакетов драйверов / `analytics-pack`, листинг в маркетплейсе, поддержка SLA |

## Учебная программа (проект)

### Ассоциированный трек (~16 ч.)

1. Объектная модель и чертежи ([object-model](object-model.md))
2. Жизненный цикл пакета ([applications](applications.md))
3. HMI оператора ([operator-guide](operator-guide.md))
4. Лабораторная работа: развертывание `demo-app` + пользовательская дашборд.

### Профессиональный трек (~24 ч.)

1. Драйверы и виртуальные устройства ([drivers](drivers.md))
2. Имитаторы SCADA ([scada](scada.md))
3. Автоматизация и рабочие процессы ([automation](automation.md), [workflows](workflows.md))
4. Лабораторная работа: Справочное руководство по SCADA или MES.

### Экспертный трек (~32 ч.)

1. Кластер и федерация ([cluster](cluster.md), [federation](federation.md))
2. Уровни историка ([historian-tiers](historian-tiers.md))
3. Агент ИИ и генератор решений ([ai-development](ai-development.md))
4. Лаборатория: готовый к работе сайт с оповещениями + приложение для оператора.

## Процесс подачи заявки

1. **Подать заявку** — контактная форма + профиль компании (юридическое лицо, регионы, вертикали).
2. **Проверка** — техническое интервью + соглашение о неразглашении.
3. **Обучение** — модули для самостоятельного обучения + лабораторные занятия под руководством инструктора (дистанционно).
4. **Экзамен** — практический сценарий развертывания (пакет + пользовательский интерфейс оператора + один артефакт автоматизации).
5. **Сертификация** — бейдж + внесение в каталог партнеров; продление ежегодно.

## Сценарии экзамена (схема)

| Уровень | Сценарий | Критерии прохождения |
|-------|----------|---------------|
| Партнер | Разверните пакет `lab-training`, добавьте виджет дашборда | Приложение оператора загружается; виджет показывает реальную стоимость |
| Профессиональный | SNMP или виртуальное устройство + правило оповещения | Драйвер БЕГЕТ; тревожные пожары на пороге |
| Эксперт | Привязка Федерации + экран диспетчерской MES | Возможность чтения с удаленного устройства; BFF вызывает ОК |
| OEM | Отправьте пакет драйверов + манифест маркетплейса | Interop CI зелёный; листинг подтверждает |

## Юридические вопросы и брендинг

- Партнеры могут использовать значок **Сертифицированного партнера ISPF** для каждого уровня (TBD).
- Для пакетов Marketplace требуются подписанные метаданные манифеста и лицензии ([marketplace](marketplace.md)).
- Пакеты драйверов GPL/LGPL исключены из профиля продукта по умолчанию до тех пор, пока не будет проведена юридическая проверка со стороны клиента.
