# CI-тестовая сортировка (S20-04)

Политика для Playwright, интеграционных тестов Gradle и шлюзов загрузки при снижении уровня успешности CI.

## Серьезность

| Уровень | Определение | соглашение об уровне обслуживания |
| ----- | ---------- | --- |
| **Чешуйка P0** | Блоки `main` объединяются или ночами >2 последовательных запуска | Исправьте или поместите в карантин в течение **24 часов** |
| **Чешуйка P1** | Сбой >1 раз в неделю, существует обходной путь | Исправление в течение **1 спринта** |
| **Чешуйка P2** | Редко (<1× мес) | Отставание |

## Поток сортировки

1. **Обнаружить** — невыполненное задание в GitHub Actions; проблема с этикеткой `ci-flake`.
2. **Назначить владельца** — владелец модуля (бэкенд/веб-консоль/развертывание).
3. **Воспроизвести** — локальная команда из [TESTING.md](testing.md) или журнала рабочего процесса.
4. **Исправить или поместить в карантин:**
   - **Предпочтительно исправление:** детерминированное ожидание, изоляция, очистка тестовых данных.
   - **Карантин:** `@Disabled` / `test.skip` со ссылкой на выпуск + срок действия (максимум 14 дней).
5. **Проверка** — зелёный на PR + один ночной полный запуск.
6. **Журнал** — примечание в [ACCELERATION_PROGRAM.md § журнал системы показателей](acceleration-program.md).

## Правила карантина

- Тесты, помещенные в карантин, **не должны** проводиться в `pr-fast`; может запускаться ночью с `continue-on-error`, только если явно отмечено как экспериментальное.
- Каждая карантинная запись в коде:

```java
@Disabled("ci-flake #1234 — quarantine until 2026-07-20")
```

```typescript
test.skip(true, "ci-flake #1234 — quarantine until 2026-07-20");
```

- Карантин с истекшим сроком действия не исправлен → перейти на уровень P0.

## CI-профили

| Рабочий процесс | Триггер | Область применения |
| -------- | ------- | ----- |
| [ci.yml](../.github/workflows/ci.yml) | PR | pr-fast: модуль + сборка, фильтры путей, без загрузки/e2e |
| [ci-nightly.yml](../.github/workflows/ci-nightly.yml) | нажмите `main`, ежедневный cron | Полное: load gates, Playwright, владение кластером |
| [load-test.yml](../.github/workflows/load-test.yml) | ночной хрон | Только загрузочные шлюзы JVM |
| [e2e-live.yml](../.github/workflows/e2e-live.yml) | еженедельно + рассылка | Прод дым (секреты) |

## Известные горячие точки (базовый показатель на июль 2026 г.)

| Площадь | Симптом | смягчение последствий |
| ---- | ------- | ---------- |
| Нагрузочные тесты в PR | Медленный + H2/Flyway в Windows | Перенесено на ночной режим (S20-01) |
| Предварительный просмотр Playwrightа | Время просмотра в мобильной версии | Только ночью |
| Туннель Федерации ИТ | Тайм-аут подключения WS/очистки буфера на медленных бегунах | `@Isolated`, бюджеты в `FederationIntegrationTestSupport` (S27); каждую ночь, если &gt;2×/неделю |

## Метрики

Отслеживайте в еженедельной системе показателей:

- Уровень успеха CI (14d) — целевой показатель ≥95%
- Open `ci-flake` issues count
- Количество тестов на карантине (должно стремиться к 0)

Collect baseline: `python tools/acceleration/collect-baseline.py`
