# Включение ISPF в реестр российского ПО

> **Статус:** Internal — Правообладатель / RU market. Теги: [doc-status](../en/doc-status.md).

Руководство для правообладателя: подготовить, как собрать техпакет и подать заявку на [reestr.digital.gov.ru](https://reestr.digital.gov.ru).

**Не является юридической консультацией.** Перед подачей согласуйте пакет с юристом и при необходимости с консультантом по реестру.

См. также: [product](product.md), [deployment](deployment.md), [license-compliance](license-compliance.md).

---

## 1. Зачем это нужно

| Эффект | Основание |
|--------|-----------|
| Преференции в госзакупках (национальный режим) | ПП РФ № 1875, реестровая запись |
| Освобождение от НДС при реализации ПО | при статусе аккредитованной IT-компании + реестр ПО |
| Размещение на маркетплейсе Минцифры | [reestr.digital.gov.ru](https://reestr.digital.gov.ru) |
| Статус «доверенного ПО» (для закупок КИИ) | ФЗ о КИИ, реестровая запись соответствует требованиям |

## 2. Нормативная база (актуально на 2026 год)

| Документ | Роль |
|----------|------|
| [ПП РФ № 1236](https://www.consultant.ru/document/cons_doc_LAW_189116/) (ред. ПП № 1937) | Правила ведения реестра, требования к заявителю и ПО |
| [Приказ Минцифры № 486](https://www.consultant.ru/document/cons_doc_LAW_366474/) | Классификатор программ (код класса в заявке) |
| [ПП РФ № 1937](https://www.consultant.ru/document/cons_doc_LAW_520975/) от 28.11.2025 | Совместимость с ≥2 проверенными ОС; отмена ПП № 325 |
| ПП РФ № 325 (2017) | **Не применяется** с 01.03.2026 — требования к офисному ПО не применяются к SCADA |

### Сроки совместимости с доенными ОС (п. 6 ПП № 1937)

Для класса **09.04** (АСУ ТП / SCADA) — **с 01.01.2028** необходима надежность работы **не менее чем на 2 ОС**, соответствующих требованиям к проверенному ПО.

Заявку на эту дату можно подать без протоколов; после включения — вынести данные о совместимости, иначе возможно внести изменения в реестр.

## 3. Класс ПО для ISPF

| Поле | Значение |
|------|----------|
| Раздел классификатора | 08 — Промышленное ПО / 09 — Средства управления процессами организации |
| **Основной класс** | **09.04** — Средства управления технологиями процессами (АСУ ТП, SCADA) |
| Формулировка класса | ПО для автоматизации управления технологиями оборудования на промышленных предприятиях |
| ОКПД 2 (из классификатора) | 62, 58.29.2, 58.29.3, 58.29.32, 58.29.4, 62.01.2, 62.01.29 |
| Дополнительный класс (опционально) | **04.08** — Интегрированные платформы для создания приложений |

### Иностранный аналог для экспертиз

Укажите в заявке продукт для сравнения сравнительной характеристики, например:

- Индуктивная автоматика **Зажигание**
- АВЕВА / Wonderware
- Ситект / Шнайдер Электрик

Сопоставление: дерево объектов, драйверы, HMI, историк, тревоги, рабочий процесс, отчёты — см. [product](product.md).

## 4. Требования к заявителю (организации)

Перед подачей проверьте по [Правилам ПП № 1236](https://www.consultant.ru/document/cons_doc_LAW_189116/), п. 5:

| № | Требование | Действие |
|---|------------|----------|
| 1 | Правообладатель — российское юрлицо / гражданин РФ | Оформить исключительные права на код в РФ-юрлица |
| 2 | Контроль РФ: >50% голосов/долей у граждан РФ или РФ-субъектов | Выписка ЕГРЮЛ, структура владения |
| 3 | Исключительные права на ПО у заявителя | Трудовые договоры + служебные произведения **или** договоры отчуждения / лицензии от авторов |
| 4 | Ограничение выплат иностранным лицензиям | Справка о доле бесплатных выплат в доходе (лимиты — в Правилах) |
| 5 | Продукт существует и доступен | Рабочий дистрибутив + страница на сайте правообладателя |

**Льготы IT-компании** (НДС, аккредитация) — внести реестр Минцифры; не путать с реестром **программ**.

## 7. Дистрибутив для экспертиз

Эксперт должен **развернуть и проверить** работоспособность. Подготовьте один из вариантов.

### Вариант А — Docker (рекомендуется)

| Артефакт | Как получить |
|----------|--------------|
| `ispf-server.jar` | `./gradlew :packages:ispf-server:bootJar -x test "-Pversion=<ver>"` |
| `web-console.zip` | `cd apps/web-console && npm ci && npm run build` → zip из `dist/` |
| Driver packs | `./gradlew syncAllDriverPacks` → `build/driver-packs/` (profile `permissive`) |
| Compose | `deploy/docker-compose.prod-stack.yml` + `deploy/air-gap-images.env` |
| Инструкция | `07-expert-review/deployment-for-expert.pdf` |

Минимальный сценарий: `bash deploy/prod-quickstart.sh` (см. [deployment](deployment.md)).

### Вариант Б — Архив с воздушным зазором

См. [air-gap-deployment](air-gap-deployment.md) — один tar с образами, JAR, UI, скриптами.

### Системные требования (для system-requirements.pdf)

| Компонент | Минимум | нравится |
|-----------|---------|---------------|
| Серверный API | 4 виртуальных ЦП, 8 ГБ ОЗУ, диск 50 ГБ | 8 виртуальных ЦП, 16 ГБ ОЗУ, твердотельный накопитель |
| ОС сервера | Linux x86_64 (Ubuntu 22.04+, Astra, РЕД ОС — по матрице §8) | Убунту 24.04 ЛТС |
| JRE | JDK/JRE 25 (в Docker-образе) | 25 |
| СУБД | PostgreSQL 16 + TimescaleDB | Отдельный кластер PG |
| Клиент HMI | Современный браузер (Chromium, Firefox, «Яндекс Браузер») | 1920×1080 |
| Сеть | TCP 8080 (api.md), 443 (HTTPS UI), доступ к устройствам по протоколам драйверов | Завершение TLS на nginx |

## 8. Сценарий приборов для эксперта (demo-scenario.pdf)

Пошаговый чеклист (~45 минут):

1. **Развёртывание** — `GET /actuator/health` → `UP`, `GET /api/v1/info` → версия.
2. **Вход** — Веб-интерфейс, роль `admin`.
3. **Дерево объектов** — открыть `root.platform.devices` (fixtures или импорт `examples/lab-training/bundle.json`).
4. **Телеметрия** — live-значения на дашборде/мемосхеме (WebSocket).
5. **Историк** — график, изменяющийся за период.
6. **Тревога** — сработать нарушение оповещения, надежность (подтверждение).
7. **Управление** — запись в устройство или вызов функции с HMI.
8. **Отчёт** — экспорт PDF или XLSX из Report Builder.
9. **Рабочий процесс** — запуск BPMN, задача в рабочей очереди.
10. **Мимика SCADA** — открыть мнемосхему, привязку к переменной.
11. **Роли** — вход как `operator`, убедитесь, что админ-редакторы включены.

Импортное лабораторное обучение:

```http
POST /api/v1/platform/packages/import?packageId=lab-training
Content-Type: application/json

<examples/lab-training/bundle.json>
```

## 10. Подача заявок на reestr.digital.gov.ru

1. Учётная запись **юридического лица** на [Госуслугах](https://www.gosuslugi.ru).
2. **КЭП** руководителя (или уполномоченного).
3. [reestr.digital.gov.ru](https://reestr.digital.gov.ru) → «Подать заявление».
4. Тип: **реестр российского ПО**.
5. Название ПО (как в маркетинге, так и в дистрибуции).
6. Класс **09.04** (+ при необходимости 04.08).
7. Загрузить техпакет (или ссылки на страницу продукта — по форме).
8. Сведения о выплатах по лицензиям, гарантиям, техподдержке, прогрессу.
9. Подтверждение права на распространение.
10. Отправить → будет по запросу экспертного заключения в срок.

**Срок рассмотрения:** до **45 рабочих дней** с даты регистрации заявления.

## 11. Доверенные ОС (до 01.01.2028)

Для класса 09.04 подготовьте **08-trusted-os/**:

| Шаг | Действие |
|-----|----------|
| 1 | Выбрать 2 ОС из реестра доверенного ПО (разные правообладатели), напр. **Astra Linux SE** + **РЕД ОС** |
| 2 | Развернуть Руководство по установке ISPF на каждую ОС |
| 3 | Выполнить сценарий §8, зафиксировать скриншоты и логи |
| 4 | Оформить протокол проверки (дата, версия ISPF, версия ОС, результат по каждому пункту §8) |
| 5 | Внести сведения в реестровую запись (подп. «ф» п. 4 Правила) |

Текущее состояние репозитория: документированный запуск на **Ubuntu/Linux + Docker**; протоколы на отечественных ОС **нужно настроить и добавить**.

## 12. Мастер-чеклист

### Юридическое

- [ ] Российское юрлицо — правообладатель
- [ ] >50% контроля в РФ
- [ ] Договоры с разработчиками / служебные задания
- [ ] Регистрация в Роспатенте (рекомендуется)
- [ ] Справка о доле иностранных выплат

### Продукт

- [ ] Стабильная версия (без SNAPSHOT в заявке)
- [ ] Класс **09.04** согласование с описанием
- [ ] Сравнение с иностранным аналогом
- [ ] Страница товара на сайте правообладателя

### Техпакет (§6)

- [ ] 01-legal — копии
- [ ] 02-продукт — описание и функции
- [ ] 03-установка — установка, требования
- [ ] 04-операция — админ, оператор, безопасность
- [ ] 05-api — обзор API
- [ ] 06-drivers — каталог и лицензии зависимостей
- [ ] 07-эксперт-обзор — дистрибутив + сценарий

### Экспертиза

- [ ] Дистрибутив разворачивается «с нуля» по инструкции
- [ ] Демо-сценарий воспроизводства происходит без доработок
- [ ] Контактные техподдержки для запросов Минцифры

### После включения

- [ ] Ежегодное до **1 июня** (изменения, выручка)
- [ ] **08-trusted-os** до **01.01.2028**
- [ ] Обновление реестровой записи при смене мажорной версии
