# руководитель оператора

Руководство для пользователей с ролью **оператор** — мониторинг оборудования, выполнение задач из рабочей очереди, просмотр событий и отчётов.

Обзор продукта: [PRODUCT.md](product.md). Технические детали пользовательского интерфейса: [WEB_CONSOLE.md](web-console.md).

---

## Вход в систему

1. Откройте веб-консоль: `http://<host>:5173`
2. Войдите под учётную запись оператора (демо: `operator` / `operator`).
3. После входа откроется **Operator HMI** — полноэкранный режим без деревьев и редакторов.

Если у вас роль администратора, но нужен операторский режим:

```
http://<host>:5173?mode=operator
```

Для подробностей приложения:

```
http://<host>:5173?mode=operator&app=platform
```

### Автозапуск приложения

Администратор может настроить для вашей учетной записи **автозапуск**: после входа сразу открывается нужное приложение оператора (например, `platform` или отраслевое приложение).

## Интерфейс оператора HMI

```
┌─────────────────────────────────────────────────────────────┐
│  [App title]                              [Logout]          │
├───────────────────────────────────────┬─────────────────────┤
│                                       │  Work Queue         │
│         Dashboard (read-only)         │  ─────────────      │
│         Виджеты: значения,            │  Event Journal      │
│         графики, таблицы              │                     │
│                                       │                     │
└───────────────────────────────────────┴─────────────────────┘
```

### Что доступно оператору

| Действие | Доступно |
|----------|:--------:|
| Просмотр дашбордов и оценок | ✓ |
| Обновление данных (в реальном времени через WebSocket) | ✓ |
| Вызов функций (кнопки на приборной панели) | ✓ |
| Очередь работ: заявить/завершить | ✓ |
| Журнал событий | ✓ |
| Редактирование макета, объектов, рабочего процесса | ✗ |
| Управление пользователями | ✗ |
| Развертывание приложений | ✗ |

## Работа с дашбордами

### Навигация между экранами

Если приложение содержит несколько дашбордов, они отображаются как **вкладки** в верхней части экрана. Переключайте вкладки для перехода между экранами (например, «Обзор», «Детали», «Отчёты»).

Виджет **dashboard-link** на экране может открыть другую панель — в текущей вкладке или в модальном окне.

### Виджеты на экране

| Виджет | Что показывает | Действия оператора |
|--------|----------------|-------------------|
| **значение** / **индикатор** | Текущее переменное значение | Только просмотр |
| **диаграмма** / **спарклайн** | График тренда | Только просмотр |
| **датчик** | Шкала | Только просмотр |
| **таблица объектов** | Таблица объектов | Кликните по строке → выбор для других виджетов |
| **карточная сетка** | Карточки объектов | Клик → выбор |
| **функциональная кнопка** | Кнопка действия | Нажатие → вызов функции объекта |
| **рабочая очередь** | Список задач BPMN | Заявить / Завершить |
| **журнал событий** | Журнал событий | Просмотр, фильтр |
| **spreadsheet** | Сетка A1 с формулами | Ввод значений и формул (если `editable`) |
| **текст** / **изображение** | Статический контент | — |

Полный справочник всех виджетов (что делает настройку, примеры): **[WIDGETS.md](widgets.md)**.

На экране с таблицей устройств (виджет `object-table` с `selectionKey`) кликните по строке, обновляет связанные виджеты — графики и индикаторы отображают данные **выбранного** устройства.

### Электронная таблица

Виджет **электронная таблица** — таблица с адресацией ячеек как в Excel (A1, B2, …). Значения и формулы пересчитываются на экране без обращения к серверу (кроме live-ячееек **привязка**).

Подробная инструкция: [SPREADSHEET_WIDGET.md](spreadsheet-widget.md).

**Кратко:**

| Элемент | Назначение |
|---------|------------|
| Формула Строка | Адрес выделенной ячейки слева; справа — значение или формула для редактирования |
| Сетка | Отображаемые результаты; в режиме **бесплатно** — любая ячейка редактируемая (кромовый переплет) |
| Панельные инструменты | В режиме **бесплатно**: отмена/повтор, копирование/вставка, экспорт CSV |

**Выделение и ввод (режим бесплатный):**

1. **Клик** по ячейке — выделение, правило в формуле строки.
2. **Двойной клик** или **F2** — редактирование прямо в ячейке.
3. **Строка формулы** — ввод чисел (`10`), текста или формулы (`=A1+B2`); **Ввести** — сохранить.
4. **Tab** / **Shift+Tab** — следующая / предыдущая ячейка; **стрелки** — перемещение по сетке.

**Горячие клавиши (бесплатно, когда фокус на таблице):**

| Клавиши | Действие |
|---------|----------|
| F2 | Редактировать ячейку |
| Войти | Открытое редактирование / после ввода — вниз |
| Выход | Отменить редактирование |
| Ctrl+Z / Ctrl+Y | Отмена / повтор |
| Ctrl+C / Ctrl+V | Копировать / вставить ячейку |

**Сохранение:** вводимые значения и формулы определяются автоматически (сессия дашборда или переменный объект — как настроен разработчик). После нажатия **F5** данные будут отображаться, если виджет изменится на переменную (`persistMode: variable`).

**Лабораторный пример:** дашборд `root.platform.dashboards.lab-calculator` — шаблонный калькулятор (режим **настроен**): редактируются только ячейки A2 и B2 суммы, пересчитываются сами.

### Динамический выбор объекта

На экранах с таблицей устройств (виджет `object-table` с `selectionKey`) кликните по строке, обновляет связанные виджеты — графики и индикаторы отображают данные **выбранного** устройства.

## Рабочая очередь (очередь задач)

Work Queue — задачи из BPMN **пользовательские задачи**, назначенные операторам.

### Типичный процесс

1. Рабочий процесс запускается (вручную администратором или по событию).
2. Процесс доходит до задачи пользователя → проблема появления в рабочей очереди.
3. Оператор нажимает **Claim** — проблема замыкается за ним.
4. Действия оператора (например, подтверждение на дашборде).
5. Нажимает **Завершено** — рабочий процесс продолжается.

### Где смотреть

- **Боковая панель** справа в HMI оператора — панель «Очередь работ».
- Виджет **рабочей очереди** на приборной панели — встроенная очередь на экран.

### Если задача недоступна

- Задача уже **востребована** другим оператором — дождитесь выполнения или обратитесь к администратору.
- Workflow в статусе `STOPPED` — новые задачи не создаются.

## Журнал событий

Боковая панель **Журнал событий** показывает последние события платформы:

- Превышение порогового датчика
- Правило оповещения «Срабатывание»
- Ручной пожар
- Системные уведомления

Уровни: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`.

Виджет **event-log** на дашборде может фильтровать события по конкретному объекту.

## Отчёты

SQL-отчёты прикладных приложений доступны через BFF или встроенные экраны приложения оператора. Формат экспорта — CSV.

Пример (если настроено приложение `demo`):

- Экран «чётОты» в приложении оператора.
- Или прямой вызов через API (только для интеграторов)

Подробнее: [REPORTS.md](reports.md).

##Частные вопросы

### Данные не обновляются

- Проверьте, что API-сервер запущен (`GET /actuator/health`).
- Обновите страницу — WebSocket переподключается автоматически.
- Убедитесь, что драйвер устройства находится в состоянии `RUNNING` (это настраивается администратором).

### Нет нужного дашборда/вкладки

Обратиться к администратору — дашборды и интерфейс оператора настраиваются в консоли администратора → `root.platform.operator-apps`.

### Ошибка при вызове функции

- Возможно, прав недостаточно — оператор может включить функцию, но не все функции доступны для оператора.
- Проверяйте события журнала и сообщайте администратору текст ошибки.

### Как вернуться в консоль администратора

Только пользователи с ролью **admin** могут переключаться:

```
http://<host>:5173?mode=admin
```

## Агент оператора (второй пилот с искусственным интеллектом)

В HMI оператора доступен **встроенный AI-ассистент** — второй пилот только для чтения для смены: тренды, отчёты, журнал событий, последовательность задач. Не изменяет конфигурацию платформы.

Подробнее: [AI_DEVELOPMENT.md](ai-development.md), BL-179.

### Как открыть

- Панель чата в HMI оператора (значок ассистента).
- URL тот же, что у operator app: `?mode=operator&app=<appId>`.

Администратор задает инструкции и загружает документы в **приложение оператора** (`root.platform.operator-apps` → Инструкции для агента / База знаний).

### Язык

Ассистент отвечает **на языке оператора** (русский/английский). Память приложения (`remember_app_memory`) автоматически использует заголовки ru/en в подсказках.

### Доступные инструменты (только чтение)

| Инструмент | Назначение |
|------------|------------|
| `get_operator_scope` | Разрешённые префиксы путей приложения |
| `list_objects` / `get_object` / `search_objects` | Обзор объектов в scope |
| `list_variables` / `describe_variables` | Текущие значения и схема |
| `get_variable_history` / `get_variable_trend` | Historian и агрегаты |
| `list_events` / `list_event_catalog` | Журнал и каталог событий |
| `list_reports` / `run_report` | Отчёты, разрешённые для приложения |
| `invoke_bff` / `invoke_tree_function` | Операционные функции (только чтение данных) |
| `list_work_queue` | Открытые задачи BPMN |
| `list_app_memory` / `remember_app_memory` | Долговременная память смены (глоссарий, нормы) |
| `list_app_documents` / `read_app_document` / `search_app_documents` | Загруженные SOP/инструкции |
| `get_operator_link` | Ссылка на дашборд или отчёт в HMI |
| `get_dashboard_layout` / `list_automation` | Просмотр layout и automation (без записи) |

**Запрещено в режиме оператора:** `create_object`, `configure_driver`, `import_package`, `configure_alert`, `set_dashboard_layout` и любые инструменты мутации.

### Типичные запросы

| Запрос операции | Что делает агент |
|------------------|------------------|
| «Какая температура на линии 2?» | `list_variables` → ответ с текущим значением |
| «Покажи тренд давления за сутки» | `get_variable_trend` |
| «Запусти сменный отчёт» | `list_reports` → `run_report` → краткое резюме цифр |
| «Что в очереди задач?» | `list_work_queue` |
| «Запомни: GPU-03 — линия 2» | `remember_app_memory` |

### Память приложения

- Общая для всех операторов одного `appId`.
- Типы: `fact`, `glossary`, `preference`, `playbook`, `correction`.
- Фразы «запомни…», исправления — агент сохраняет через `remember_app_memory`.
