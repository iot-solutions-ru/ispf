# Каталог ISA-95 — сопоставление дерева объектов ISPF (BL-124).

Как объекты платформы ISPF сопоставляются с иерархией оборудования **ANSI/ISA-95** (IEC 62264) и действиями MES. Используйте это при разработке приложений, пакетов и документации для клиентов-производителей.

**См. также:** [OBJECT_MODEL.md](object-model.md), [REFERENCE_MES_OEE_WALKTHROUGH.md](reference-mes-oee-walkthrough.md), [mes-oee-reference](../examples/mes-oee-reference/).

---

## Уровни ISA-95 → ISPF

| уровень ISA-95 | Имя | Конструкция ISPF | Пример пути |
|--------------|------|----------------|--------------|
| **Уровень 4** | Бизнес-планирование и логистика | Отчеты, приложения SQL, внешняя синхронизация ERP | `root.platform.reports.*`, пакеты приложений с BFF + исходящие |
| **Уровень 3** | Производственные операции (MES) | Рабочие процессы, очередь работ, OEE BFF, схемы приложений | `root.platform.workflows.*`, `mes_oee_*` функции, `app_mes_*` таблицы |
| **Уровень 2** | Диспетчерское управление (SCADA/HMI) | Панели мониторинга, мнемосхемы, правила оповещений, корреляторы | `root.platform.dashboards.*`, `root.platform.mimics.*`, `root.platform.alert-rules.*` |
| **Уровень 1** | Базовый контроль | Устройства, драйверы, переменные, привязки | `root.platform.devices.*`, среда выполнения драйвера, точки MQTT/OPC/BACnet |
| **Уровень 0** | Физический процесс | Представлено косвенно через экземпляры устройств и семантические теги | Переменные устройства, метаданные Brick/Haystack на объектах |

ISPF **не** обеспечивает фиксированное дерево папок ISA-95 — сопоставление является **традиционным** посредством именования, схем элементов и пакетов приложений.

## Иерархия оборудования (физическая модель)

Рекомендуемое соглашение о точечных путях для проектов MES/SCADA:

```
root.platform.instances.{site}
  └── .areas.{area}
        └── .lines.{line}
              └── .units.{unit}          ← work center / machine
                    └── devices.{tag}    ← Level 1 device object
```

| Срок ISA-95 | Тип узла ISPF | Заметки |
|-------------|----------------|-------|
| Предприятие/Сайт | `INSTANCES` или корневой объект приложения | Один сайт на каждого арендатора в мультитенантных системах (BL-125+) |
| Area | Child under site instance | e.g. `...instances.plant-a.areas.refining` |
| Production line | Child under area | Links to OEE shift rows (`line_code`) |
| Рабочий блок / оборудование | `DEVICE` или напечатанный экземпляр | Драйверные точки, функции, привязки мнемосхем |
| Material lot / order | App SQL schema + BFF | e.g. `mes-reference` work orders |

Пример из **mes-oee-reference**: код строки `LINE-A01` в SQL сопоставляется с логической строкой; Концентратор устройств `demo-sensor-01` содержит функции BFF (шаблон для небольших демонстраций).

## Деятельность MES (Уровень 3)

| Деятельность ISA-95 | Механизм ISPF |
|-----------------|----------------|
| Планирование производства | Приложение SQL + пользовательский интерфейс оператора/отчеты |
| Диспетчеризация | Пользовательские задачи рабочего процесса → очередь работ |
| Performance analysis (OEE) | BFF `mes_oee_getKpi`, dashboards |
| Техническое обслуживание/отслеживание качества | События + архиватор + таблицы приложений |
| Управление тревогами/исключениями | Правила оповещений → корреляторы → рабочие процессы |

Справочные руководства:

- OEE: [REFERENCE_MES_OEE_WALKTHROUGH.md](reference-mes-oee-walkthrough.md)
– Эскалация: [REFERENCE_ESCALATION_TEMPLATES.md](reference-escalation-templates.md)
- Приказы MES: [mes-reference](../examples/mes-reference/)

## Информационная модель (переменные и семантика)

| Концепция | ИСФФ |
|---------|------|
| Данные сегмента обработки | Переменные устройства, архиватор |
| Возможности оборудования | Функции объекта + метаданные |
| Personnel / role | `root.platform.security.users`, workflow `assigneeRole` |
| Определение материала | Схема приложения (пакетная миграция) |
| Семантическая маркировка | Панели «Кирпич/Стог сена» на объектах (БЛ-104) |

## Краткий контрольный список для разработчиков решений

1. **Участок/Область/Линия** — модель в иерархии `instances` или приложения; сохраняйте стабильность путей устройства для привязок.
2. **HMI уровня 2** — одна панель мониторинга на роль оператора; симулирует привязку к устройствам уровня 1.
3. **Level 3 MES** — рабочие процессы для исключений; BFF/SQL для KPI; корреляторы повторяющихся событий.
4. **Уровень 4** — отчеты и экспорт (шаблоны исходящих сообщений CSV/PDF/1C в эталонных приложениях).
5. **Документация** — ссылайтесь на этот каталог в документации по решениям для клиентов; расширить с помощью таблицы путей для конкретного проекта.

## Сопутствующие стандарты

| Стандарт | Поддержка ISPF |
|----------|--------------|
| ИСА-95/МЭК 62264 | Этот каталог (концептуальное картографирование) |
| ИСА-88 (серия) | Частичный — рабочие процессы + состояние приложения; нет пакетного двигателя |
| Кирпич / Стог сена | Семантические метаданные, API запросов (BL-101–105) |
| BPMN 2.0 | Subset in `ispf-plugin-workflow` |
