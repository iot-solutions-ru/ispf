# Руководство оператора

Руководство для пользователей с ролью **operator** — мониторинг оборудования, выполнение задач из work queue, просмотр событий и отчётов.

Обзор продукта: [PRODUCT.md](PRODUCT.md). Технические детали UI: [WEB_CONSOLE.md](WEB_CONSOLE.md).

---

## Вход в систему

1. Откройте Web Console: `http://<host>:5173`
2. Войдите под учётной записью оператора (демо: `operator` / `operator`).
3. После входа откроется **Operator HMI** — полноэкранный режим без дерева объектов и редакторов.

Если у вас роль admin, но нужен operator-режим:

```
http://<host>:5173?mode=operator
```

Для конкретного приложения:

```
http://<host>:5173?mode=operator&app=platform
```

### Автозапуск приложения

Администратор может настроить для вашей учётной записи **auto-start**: после входа сразу открывается нужное operator app (например, `platform` или отраслевое приложение).

```
┌─────────────────────────────────────────────────────────────┐
│  [App title]                              [Logout]          │
├───────────────────────────────────────┬─────────────────────┤
│                                       │  Work Queue         │
│         Dashboard (read-only)         │  ─────────────      │
│         Виджеты: значения,            │  Event Journal      │
│         графики, таблицы              │                     │
│                                       │                     │

```

### Что доступно оператору

| Действие | Доступно |
|----------|:--------:|
| Просмотр дашбордов и значений | ✓ |
| Обновление данных (live через WebSocket) | ✓ |
| Вызов функций (кнопки на дашборде) | ✓ |
| Work queue: claim / complete | ✓ |
| Журнал событий | ✓ |
| Редактирование layout, объектов, workflow | ✗ |
| Управление пользователями | ✗ |
| Deploy приложений | ✗ |

## Работа с дашбордами

### Навигация между экранами

Если приложение содержит несколько дашбордов, они отображаются как **вкладки** в верхней части экрана. Переключайте вкладки для перехода между экранами (например, «Обзор», «Детали», «Отчёты»).

Виджет **dashboard-link** на экране может открывать другой дашборд — в текущей вкладке или в модальном окне.

### Виджеты на экране

| Виджет | Что показывает | Действия оператора |
|--------|----------------|-------------------|
| **value** / **indicator** | Текущее значение переменной | Только просмотр |
| **chart** / **sparkline** | График тренда | Только просмотр |
| **gauge** | Шкала | Только просмотр |
| **object-table** | Таблица объектов | Клик по строке → выбор для других виджетов |
| **card-grid** | Карточки объектов | Клик → выбор |
| **function-button** | Кнопка действия | Нажатие → вызов функции объекта |
| **work-queue** | Список задач BPMN | Claim / Complete |
| **event-log** | Журнал событий | Просмотр, фильтр |
| **spreadsheet** | Сетка A1 с формулами | Ввод значений и формул (если `editable`) |
| **text** / **image** | Статический контент | — |

Полный справочник всех виджетов (что делает каждая настройка, примеры): **[WIDGETS.md](WIDGETS.md)**.

На экранах с таблицей устройств (виджет `object-table` с `selectionKey`) клик по строке обновляет связанные виджеты — графики и индикаторы показывают данные **выбранного** устройства.

### Электронная таблица (spreadsheet)

Виджет **spreadsheet** — таблица с адресацией ячеек как в Excel (A1, B2, …). Значения и формулы пересчитываются на экране без обращения к серверу (кроме live-ячееек **binding**).

Подробная инструкция: [SPREADSHEET_WIDGET.md](SPREADSHEET_WIDGET.md).

**Кратко:**

| Элемент | Назначение |
|---------|------------|
| Строка формул | Адрес выделенной ячейки слева; справа — значение или формула для редактирования |
| Сетка | Отображаемые результаты; в режиме **free** — любая ячейка редактируема (кроме binding) |
| Панель инструментов | В режиме **free**: отмена/повтор, копирование/вставка, экспорт CSV |

**Выделение и ввод (режим free):**

1. **Клик** по ячейке — выделение, содержимое в строке формул.
2. **Двойной клик** или **F2** — редактирование прямо в ячейке.
3. **Строка формул** — ввод числа (`10`), текста или формулы (`=A1+B2`); **Enter** — сохранить.
4. **Tab** / **Shift+Tab** — следующая / предыдущая ячейка; **стрелки** — перемещение по сетке.

**Горячие клавиши (free, когда фокус на таблице):**

| Клавиши | Действие |
|---------|----------|
| F2 | Редактировать ячейку |
| Enter | Открыть редактирование / после ввода — вниз |
| Esc | Отменить редактирование |
| Ctrl+Z / Ctrl+Y | Отмена / повтор |
| Ctrl+C / Ctrl+V | Копировать / вставить ячейку |

**Сохранение:** введённые значения и формулы сохраняются автоматически (сессия дашборда или переменная объекта — как настроил разработчик). После **F5** данные остаются, если виджет привязан к переменной (`persistMode: variable`).

**Lab-пример:** дашборд `root.platform.dashboards.lab-calculator` — шаблонный калькулятор (режим **configured**): редактируются только ячейки A2 и B2, суммы пересчитываются сами.

### Динамический выбор объекта

На экранах с таблицей устройств (виджет `object-table` с `selectionKey`) клик по строке обновляет связанные виджеты — графики и индикаторы показывают данные **выбранного** устройства.

## Work Queue (очередь задач)

Work Queue — задачи из BPMN **user task**, назначенные операторам.

### Типичный процесс

1. Workflow запускается (вручную администратором или по событию).
2. Процесс доходит до user task → задача появляется в Work Queue.
3. Оператор нажимает **Claim** — задача закрепляется за ним.
4. Оператор выполняет действие (например, подтверждает на дашборде).
5. Нажимает **Complete** — workflow продолжается.

### Где смотреть

- **Sidebar** справа в Operator HMI — панель Work Queue.
- Виджет **work-queue** на дашборде — встроенная очередь на экране.

### Если задача недоступна

- Задача уже **claimed** другим оператором — дождитесь завершения или обратитесь к администратору.
- Workflow в статусе `STOPPED` — новые задачи не создаются.

## Журнал событий

Sidebar **Event Journal** показывает последние события платформы:

- Превышение порога датчика
- Срабатывание alert rule
- Ручной fire события
- Системные уведомления

Уровни: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`.

Виджет **event-log** на дашборде может фильтровать события по конкретному объекту.

## Отчёты

SQL-отчёты прикладных приложений доступны через BFF или встроенные экраны operator app. Формат экспорта — CSV.

Пример (если настроено приложение `demo`):

- Экран «Отчёты» в operator app
- Или прямой вызов через API (только для интеграторов)

Подробнее: [REPORTS.md](REPORTS.md).

## Частые вопросы

### Данные не обновляются

- Проверьте, что API-сервер запущен (`GET /actuator/health`).
- Обновите страницу — WebSocket переподключится автоматически.
- Убедитесь, что драйвер устройства в статусе `RUNNING` (это настраивает администратор).

### Нет нужного дашборда / вкладки

Обратитесь к администратору — дашборды и operator UI настраиваются в admin console → `root.platform.operator-apps`.

### Ошибка при вызове функции

- Возможно, недостаточно прав — оператор может вызывать функции, но не все функции разрешены для роли operator.
- Проверьте журнал событий и сообщите администратору текст ошибки.

### Как вернуться в admin console

Только пользователи с ролью **admin** могут переключиться:

```
http://<host>:5173?mode=admin
```
