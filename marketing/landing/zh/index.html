为何选择 ISPF

SCADA/MES 无需模块堆砌

OPC 服务器、historian、HMI、workflow 与 MES 逻辑 — 同一对象树节点， 统一 REST/WebSocket API。参考应用：能源 (mini-tec)、仓储 (warehouse)、MES。

经典栈 (Ignition + KEPServerEX + PI + Opcenter)

  • OPC + historian + SCADA + MES — 不同厂商与许可
  • 现场与账务集成 — 手工 ETL、标签重复
  • 定制 = 各层数月 Java/C# 与集成商
  • HMI、告警与 workflow — 不同 runtime 与 API
  • AI — 外挂式，非 tree-first 开发
  • 栈各层厂商锁定
