Почему ISPF

SCADA/MES без зоопарка модулей

OPC-сервер, historian, HMI, workflow и MES-логика — узлы одного дерева объектов с единым REST/WebSocket API. Reference apps: энергетика (mini-tec), склад (warehouse), MES.

Классический стек (Ignition + KEPServerEX + PI + Opcenter)

  • OPC + historian + SCADA + MES — отдельные вендоры и лицензии
  • Интеграция поля и учёта — ручной ETL, дублирование тегов
  • Кастомизация = месяцы Java/C# и интеграторы на каждом уровне
  • HMI, алерты и workflow — разные runtime и API
  • AI — bolt-on, не tree-first разработка
  • Vendor lock-in на каждом уровне стека
