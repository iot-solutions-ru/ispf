Три уровня автоматизации — один продукт

Классический проект: OPC-сервер + historian + SCADA + MES + интеграция. ISPF закрывает весь стек declarative-конфигурацией на дереве объектов — от полевого уровня до производственных отчётов и operator HMI.

SCADA · мнемосхемы

P&ID и однолинейные схемы — не отдельный SCADA-runtime

Мнемосхемы живут в object tree как объекты MIMIC: редактор символов, live-привязки к переменным устройств, ортогональные соединения и управление с operator HMI. Логика остаётся в дереве — схема только отображает и отправляет команды.

Каталог символов Танки, клапаны, насосы, трубопроводы, электрооборудование, датчики — ~50 SVG + custom SVG
Live bindings Уровни, состояния, тревоги, качество OPC — привязка к objectPath и selectionKey
Управление с HMI setVariable, toggle, invokeFunction, navigate — клик и контекстное меню на символе
Редактор Visio-class Палитра · Place · Connect · слои · snap · align · свойства · import/export JSON
Редактор мнемосхем

Visio-class редактор прямо в браузере

Полноэкранный редактор: палитра символов, слои, свойства, bindings и actions — без десктопного SCADA-пакета. Схема сохраняется в object tree и сразу работает на operator HMI.

  • ✓
    Библиотека символовТанки, задвижки, насосы, трубы, крестовины, сигнальные лампы — процесс и электрика
  • ✓
    Select · Place · ConnectРазмещение, ортогональные соединения по портам, smart-snap и сетка
  • ✓
    Align · Distribute · TransformВыравнивание, flip, поворот ±90°, undo/redo
  • ✓
    Панель свойствBindings, format rules, actions, слои — import/export JSON diagram

Почему ISPF

SCADA/MES без зоопарка модулей

OPC-сервер, historian, HMI, workflow и MES-логика — узлы одного дерева объектов с единым REST/WebSocket API. Reference apps: энергетика (mini-tec), склад (warehouse), MES.

Классический подход

  • OPC + historian + SCADA + MES — отдельные вендоры
  • Интеграция поля и учёта — ручной ETL и скрипты
  • Кастомизация = месяцы Java и интеграторы
  • HMI, алерты и workflow — разные миры
  • Vendor lock-in на каждом уровне

ISPF

  • IoT + SCADA + MES на одном object tree

  • Historian, алерты, BPMN — без отдельных серверов
  • Alert rules, correlators и workflow — узлы дерева
  • AGPL ядро + коммерческие bundle и driver packs
